# Roadmap

This roadmap is ordered by execution priority.

## Phase 0 - Baseline Hardening (now)

Goal: make current MVP robust before scaling users.

### P0 tasks

- Fix unanswered-question behavior (no forced "A" defaults)
- Persist timer/session safely on refresh
- Add API rate limiting for public endpoints
- Add clear error states for failed network requests
- Add simple production logging + error tracking

### Exit criteria

- No critical bug in quiz -> submit -> review -> progress flow
- Manual smoke test passes on desktop and mobile

## Phase 1 - Content Scale (200+ Quant)

Goal: have enough content depth for meaningful practice.

### P0 tasks

- Expand topic taxonomy and coverage targets
- Add item quality checklist per question
- Reach 200+ published original quant questions
- Ensure each item has EN + ES explanation

### Exit criteria

- Every major Quant topic has beginner/intermediate/hard coverage
- Validation script passes for full bank

## Phase 2 - UX and Learning Quality

Goal: improve retention and study effectiveness.

### P1 tasks

- Add exam focus mode
- Improve dashboard visual clarity
- Add "weak areas first" recommendations
- Improve loading/performance for low bandwidth users

### Exit criteria

- Better completion rate in quiz sessions
- Clear user understanding of strengths/weaknesses

## Phase 3 - Public Beta (20-50 users)

Goal: collect real learner feedback safely.

### P1 tasks

- Recruit beta testers (LatAm focus)
- Track top user issues and fix quickly
- Add lightweight support channel (form or email)

### Exit criteria

- 20-50 active testers
- No blocking bugs for core workflow

## Phase 4 - Growth and Contributor Engine

Goal: scale question supply without losing quality.

### P1 tasks

- Launch contributor call on X/Twitter and other channels
- Use intake form with authorship statement
- Review and publish pipeline with double reviewer requirement

### Exit criteria

- Consistent weekly inflow of candidate questions
- Review SLA defined and followed

## Backlog (after V1 public)

- Full GRE simulation mode
- Verbal module
- AWA module
- Better score calibration model
- Study plan generator
